# cgv-bot-2026 설계 문서

리뉴얼된 CGV.co.kr(Next.js App Router + RSC) 환경에서 동작하는 좌석 자동 예매 북마클릿.

- 작성일: 2026-04-19
- 저장소: https://github.com/H4R1B0/cgv-bot-2026
- 배포 URL: https://h4r1b0.github.io/cgv-bot-2026/
- 스크립트 URL: https://h4r1b0.github.io/cgv-bot-2026/cgv-bot.js

## 1. 배경

- 원조 `ryubro/CGV-Bot`은 jQuery 북마클릿으로, `#ticket_iframe` 내부 `$.cgv.data.SEAT_INFO`와 전역 함수(`ftSeatClickListener`, `loadSeatInfo`, `reservationSeatReset`, `ticketStepNext`)에 의존.
- 2026년 CGV 리뉴얼로 위 전역이 전부 사라짐 → 원조 북마클릿 동작 불가.
- 별도 프로젝트 `cgv-seat`(Python + Playwright CDP)에서 새 CGV의 API와 sessionStorage 구조를 완전 리버스엔지니어링 완료.
- 본 프로젝트는 `cgv-seat`의 지식 기반으로 원조의 북마클릿 UX를 리뉴얼된 CGV에서 재현한다.

## 2. 목표 / 비목표

### 목표
- CGV 좌석 선택 페이지에서 북마크 한 번 클릭으로 선호 좌석 우선순위 지정 및 자동 폴링/선점.
- 설치 이후 유저 설정/입력 불필요(제로 설정).
- 유지보수 시 북마크 재설치 없이 스크립트만 갱신 가능.

### 비목표
- 결제 자동화 (선점 이후는 사용자가 수동 진행).
- 로그인/NetFunnel 우회 (로그인은 브라우저 세션에 의존).
- 모바일 브라우저 지원 (북마클릿 특성상 PC Web 전용).

## 3. 아키텍처

### 3.1 배포 형태
- **로더 북마클릿** (약 200자): GitHub Pages 호스팅 스크립트를 주입하는 짧은 `javascript:` URL.
- **메인 스크립트** (`dist/cgv-bot.js`): 전체 기능을 담은 번들, 캐시 버스터(`?v=timestamp`) 쿼리로 즉시 갱신.
- **설치 페이지** (`dist/index.html`): 북마크 드래그용 링크와 사용법.
- **배포**: `gh-pages` npm 패키지로 `dist/`를 `gh-pages` 브랜치에 푸시.

### 3.3 데이터 흐름

```
[User Click Bookmarklet]
        ↓
[loader.js injects <script src="…/cgv-bot.js?v=…">]
        ↓
[main.js]
   ├─ 페이지 가드 (host/path/sessionStorage 확인)
   ├─ context.read() → {siteNo, scnYmd, scnsNo, scnSseq, custNo, count}
   ├─ ui.mount() → 상단 배너 + 하단 완료 버튼
   └─ priority.attach() → 좌석 버튼 클릭 리스너
        ↓
[User clicks seats in preferred order]
   → 좌석 번호 위에 1, 2, 3… 오버레이
   → 선택 수 ≥ count 시 완료 버튼 활성
        ↓
[User clicks 완료]
   → monitor.start(context, prefSeats, count)
        ↓
[polling every 7s]
   ├─ api.searchIfSeatData(context)
   ├─ priority.findBestCombo(prefSeats, available, count) — 연석 + 통로감지 + 우선순위 합 최소
   └─ 발견 시: api.seatTempPrmp(context, combo)
        ↓
[hold success]
   ├─ sessionStorage.pid = {paymNo, paymVrifyNo}
   ├─ notify.bell()
   ├─ notify.banner("🔔 결제하세요!")
   └─ notify.blinkTitle()
        ↓
[사용자가 수동으로 결제 진행]
```

## 4. 모듈별 설계

### 4.1 `context.js`
- `read()` → `{siteNo, scnYmd, scnsNo, scnSseq, custNo, count, mov, com}` 반환.
- 파싱 대상:
  - `sessionStorage.com` → `{siteNo, saleDt}` (saleDt는 폴백용).
  - `sessionStorage.mov` → `{scnYmd, scnsNo, scnSseq, custNo, sachlTypCd, prodBnduCd, movNo}`.
  - `sessionStorage.movieGoers` → 인원 합계 (`일반 + 청소년 + 경로 + 우대`) = `count`.
- 필수 필드 누락 시 예외 → `ui.showError("예매 플로우 처음부터 다시 시작하세요")`.

### 4.2 `api.js`
- Base URL: `https://api.cgv.co.kr`.
- 공통 `fetch` 래퍼: `credentials: 'include'`, JSON 파싱, `statusCode !== 0` 시 `CGVAPIError` 투척.
- 메서드:
  - `getSeats(ctx)` → `GET /cnm/atkt/searchIfSeatData?coCd=A420&siteNo=…&scnYmd=…&scnsNo=…&scnSseq=…&seatAreaNo=001&cusgdCd=01&custNo=…` → `SeatInfo[]`.
  - `holdSeats(ctx, combo)` → `POST /cnm/seatTemp/seatTempPrmp` with body:
    ```js
    {
      coCd: "A420", bymd: "", mbltNo: "",
      siteNo, scnYmd, scnsNo, scnSseq,
      movAtktNo: "", custNo, cusgdCd: "01",
      nmbrCrtfNo: "", sachlCd: "10",
      atktChnlCd: "01", sachlTypCd: "01", rtctlScopCd: "08",
      seatPrmpDataList: combo.map(s => ({
        seatRowNm, seatNo, seatLocNo, sbordNo, seatAreaNo, szoneNo
      }))
    }
    ```
  - 응답의 `data.paymNo / data.paymVrifyNo`를 `sessionStorage.pid`에 기록.
- `mbltNo`는 실제 동작에서 불필요함이 확인되어 빈 문자열로 송신.

### 4.3 `priority.js`
- 좌석 버튼 탐색: `document.querySelectorAll('button[data-seatlocno]')`.
- 메타 추출: `data-seatlocno` 값으로 API 응답의 `SeatInfo`와 매칭(매 폴링 시점에 최신화).
- 클릭 리스너: 해당 버튼에 오버레이 `<span class="cgvbot-pri">N</span>`을 얹어 우선순위 표시.
- 재클릭 시 해제 + 남은 좌석 번호 재정렬(1-based compact).
- 연석 탐색 (`findBestCombo`):
  1. 선호 좌석 중 현재 가용한 것만 남김.
  2. `row`로 그룹핑, 각 그룹 내 `num` 오름차순 정렬.
  3. 슬라이딩 윈도우 `count`로 연속 `num` 조합 수집.
  4. **통로 필터**: API 응답의 `seatLocNo[6:10]` X좌표 인접성 확인(차이 ≥ 4 → 통로 → 제외).
  5. 남은 후보 중 **우선순위 합 최소** 세트 리턴.

### 4.4 `monitor.js`
- `start(ctx, prefSeats, count)`:
  - `interval = 7000ms`, `minInterval = 5000ms` (하드코딩, 상향만 허용).
  - 루프: `getSeats` → `findBestCombo` → 발견 시 `holdSeats`.
  - `CGVAPIError` 시 현 interval × 2 백오프, 최대 60초.
  - 선점 실패 (타 유저가 먼저 잡음 등) 시 다음 조합 시도 → 다 실패하면 다음 폴링 사이클로.
  - 폴링 중 상태는 상단 배너에 "감시 중… 마지막 확인 HH:MM:SS" 표시.
  - `stop()` / 페이지 이탈 시 `clearTimeout` 정리.

### 4.5 `ui.js`
- 상단 배너 (`#cgvbot-banner`): 상태 메시지, `z-index: 99999`, 빨강/회색/초록 3단계.
- 하단 완료 버튼 (`#cgvbot-start`): 초기 비활성, 선택 수 ≥ count 시 활성.
- 좌석 우선순위 오버레이: 좌석 버튼 내부에 절대 위치 `<span>` 삽입, CSS로 원형 뱃지.
- 스타일: CSS 객체로 `Object.assign(el.style, {...})` (CSS 파일 없음, 번들 내 포함).

### 4.6 `notify.js`
- `bell()`: `new Audio('data:audio/mpeg;base64,…').play()` — 짧은 알림음 base64 임베드(네트워크 독립).
- `banner(text)`: `ui.js`의 배너 색을 초록으로 전환 + 메시지 갱신.
- `blinkTitle()`: `setInterval(500)`로 `"🔔 결제하세요!"` ↔ `origTitle` 토글, 윈도우 `focus` 시 정지.

### 4.7 `main.js` 초기화
```js
if (window.__cgvBot2026__) window.__cgvBot2026__.reset();
if (!pageGuard()) return;
const ctx = context.read();
const state = {...};
window.__cgvBot2026__ = state;
ui.mount(state);
priority.attach(state);
```

## 5. 에러 처리 / 엣지 케이스

| 상황 | 처리 |
|---|---|
| CGV 도메인/좌석 페이지 아님 | `alert` 후 조용히 종료 |
| `sessionStorage.mov` 없음 | 배너에 "예매 플로우 처음부터 다시 시작하세요" 표시 |
| 인원수 = 0 | "인원 선택 단계부터 다시 진행하세요" |
| 좌석 페이지가 좌석 구역(`seatAreaNo`) 여러 개 | v1에서는 `001`만 지원, 배너로 한계 고지 |
| 북마클릿 두 번 클릭 | `__cgvBot2026__` 확인 후 기존 UI 파괴 → 재초기화 |
| `searchIfSeatData` 연속 실패 | 백오프 후 재시도 (최대 60초 간격) |
| `seatTempPrmp` 실패 | 다음 조합 시도 → 모두 실패 시 폴링 계속 |
| NetFunnel 차단 | 폴링 중단 + 배너 "차단 감지, 잠시 후 재시도" + 60초 정지 |
| 사용자가 브라우저 탭 이탈 | 폴링은 백그라운드에서도 계속 (Chrome은 최소 동작) |

## 6. 테스트 전략

완전 자동화 테스트는 실제 CGV 세션에 의존하므로 제한적. 실용적 수준:

- **단위 테스트**(`vitest`):
  - `priority.findBestCombo` — 가짜 `SeatInfo[]` + 선호 리스트로 다양한 조합 시나리오 (연석 충분/부족/통로 가림/우선순위 동률).
  - `context.read` — `sessionStorage` 모킹(`jsdom` 또는 간이 mock)으로 필드 누락 케이스.
  - `api.js` — `fetch` mock으로 정상/`statusCode !== 0`/네트워크 오류 분기.

- **수동 테스트 체크리스트**(README에 기재):
  - 좌석 1개 단일 선택 → 선점
  - 연석 2개 (인원 2명) → 선점
  - 이미 선점된 좌석만 선택 → 폴링 계속, 배너 상태 갱신
  - 통로 사이 좌석 선택 → 탐색에서 제외되는지 확인
  - 북마크 두 번 클릭 → 중복 UI 발생하지 않음

## 7. 빌드 / 배포

### 7.1 `build.js` 책임
1. `esbuild`로 `src/main.js` → `dist/cgv-bot.js` (minify, `format: 'iife'`).
2. `loader.js`를 읽어 minify → `javascript:` URL로 변환.
3. `dist/index.html` 생성: 드래그용 앵커(`href`=북마클릿 URL) + 원조 YouTube 사용법 링크.

### 7.2 `package.json` 스크립트
```jsonc
{
  "scripts": {
    "build": "node build.js",
    "deploy": "npm run build && gh-pages -d dist",
    "test": "vitest run"
  }
}
```

### 7.3 최초 배포
1. `gh repo create H4R1B0/cgv-bot-2026 --public`.
2. GitHub Pages: Source = `gh-pages` 브랜치.
3. `npm run deploy` 1회 → `https://h4r1b0.github.io/cgv-bot-2026/` 활성화.

## 8. 향후 개선 여지 (v2 이후)

- 좌석 구역(`seatAreaNo`) 자동 탐색.
- 우선순위 UI를 DnD로 재정렬.
- 선점 TTL 정확한 값 파악 후 카운트다운 표시.
- 데스크탑 알림 (Notification API) 옵션.
- 원조 CGV-Bot 호환: 북마클릿 URL이 동일하면 재사용 가능하도록 레거시 로더 유지.
